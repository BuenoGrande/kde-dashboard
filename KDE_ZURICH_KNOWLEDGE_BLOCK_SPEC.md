# KDE Zurich Knowledge Block Specification

Prepared for: KDE - Kantonaler Deutschtest fuer die Einbuergerung, Canton of Zurich  
Access date for web sources: 2026-07-03  
Purpose: exam-specific knowledge block for curriculum tracking, lesson planning, exercise generation, readiness validation, and teacher coordination.

## 1. Executive Summary

The KDE is a Zurich cantonal German-language exam used as language proof in naturalisation contexts. It is not the separate civic-knowledge test. Official Zurich material confirms four tested skills: Hören and Sprechen at B1, Lesen and Schreiben at A2. The written block lasts 90 minutes including introduction; the oral block lasts 15-20 minutes. The public model test confirms everyday Zurich-style domains such as pharmacy directions, doctor voicemail, radio announcements about local events and road closures, Swiss-German voicemail, local festival programmes, hospital/public-service announcements, paper disposal, mushroom inspection, German courses with childcare, Spitex, dog registration, and voting logistics as reading material.

This specification treats every factual claim with a source-basis label:

| Label | Meaning |
|---|---|
| Officially confirmed | Directly stated by Canton Zurich pages or official KDE PDFs. |
| Confirmed by recognised provider | Stated by a provider listed on the official Zurich KDE language-school table. |
| Inferred from official model test | Derived from task content in the official KDE model test. |
| Reasonable preparation extrapolation | Not public as an official KDE task, but realistic Zurich everyday language aligned to official task types. |
| Unofficial / anecdotal | Useful only as weak context; no such source is used for core curriculum design. |

The curriculum is therefore not a generic Goethe/telc curriculum. It is an exam-environment map: skills, domains, vocabulary, phrase functions, scenarios, validation rules, and schema entities for a learning app.

## 3. Exam Blueprint

### 3.1 Hören

| Field | Blueprint |
|---|---|
| CEFR level | B1 - Officially confirmed |
| Duration | 15 minutes - Officially confirmed |
| Number of tasks | 4 - Officially confirmed |
| Model-test response type | Richtig/Falsch statements after one listening - Inferred from official model test |
| Official examples | Street conversation about doctor/pharmacy/directions; telephone answering machine at a practice; radio announcement about local event/road closure; Swiss-German answering-machine message about parent evening/callback. |
| Likely domains | Health, pharmacy, directions, phone/voicemail, appointments, school/family, local events, public announcements, road closures, Swiss-German gist. |
| Student must do | Identify topic, speaker, intention, true/false details, dates, times, locations, numbers, route instructions, open/closed status, callback requests, and simple Swiss-German gist. |
| Validation criteria | At least 70-80% accuracy in practice tasks; exact extraction for time/date/phone/action; can summarise who called, why, and what must happen next. |

### 3.2 Lesen

| Field | Blueprint |
|---|---|
| CEFR level | A2 - Officially confirmed |
| Duration | 35 minutes - Officially confirmed |
| Number of tasks | 4 - Officially confirmed |
| Model-test response types | Multiple choice, matching situations to notices, Ja/Nein statements - Inferred from official model test |
| Official examples | Sommerfest Fällanden programme; Stadtspital open day; notices for library story time, German course with childcare, Spitex, paper collection, mushroom inspection, emergency doctors, dog tax/registration; voting and ballot logistics. |
| Likely domains | Local events, health/public services, community services, waste/recycling, courses, elderly care, pets/dog registration, emergency services, voting logistics, admin notices. |
| Student must do | Scan for W-information, match need to correct service, identify deadlines/rules, distinguish allowed/forbidden, find opening hours and contact/action. |
| Validation criteria | Correct answer plus evidence word in text; can explain why a notice matches a situation; can identify action, deadline, place, and responsible office. |

### 3.3 Schreiben

| Field | Blueprint |
|---|---|
| CEFR level | A2 - Officially confirmed |
| Duration | 25 minutes - Officially confirmed |
| Number of tasks | 1 - Officially confirmed |
| Official model task | Write an invitation to work colleagues for an Apéro after ten years in the same company; start with date and suitable greeting; write 1-2 sentences per bullet; total about 40 words; include reason, when/where, request to register, closing and name. |
| Likely domains | Invitations, appointments, cancellations, information requests, doctor/school/office messages, neighbour/landlord messages, absence/apology, registration/confirmation. |
| Student must do | Write a complete practical text with date, greeting, reason, details, request/action, closing, name, correct register, A2 clarity. |
| Validation criteria | All task bullets included; 35-55 words unless prompt says otherwise; understandable word order; polite formula; no missing date/greeting/closing; message is actionable. |

### 3.4 Sprechen

| Field | Blueprint |
|---|---|
| CEFR level | B1 - Officially confirmed |
| Duration | 15-20 minutes total - Officially confirmed |
| Structure | Two tasks: self-introduction, then everyday-topic picture-card discussion - Officially confirmed |
| Task 1 | Candidate introduces themself about 2 minutes using prompts: name, residence, family, work, free time. |
| Task 2 | Candidate chooses one of 4-5 scenario-based picture cards and speaks about the topic about 5 minutes; examiner may ask targeted follow-up questions. |
| Language choice | Candidate may speak Standard German or Swiss German; candidate does not need to produce Swiss German if choosing Standard German. |
| Likely domains | Personal details, work, family, free time, housing, health, admin, school/family, local life, transport, shopping/services, waste/recycling, integration. |
| Student must do | Speak connectedly, answer follow-up questions, give reasons/examples, repair communication, use polite requests, describe familiar situations, keep going under examiner prompts. |
| Validation criteria | 90-120 seconds for self-introduction; 3-5 minutes for picture-card practice; answers direct questions; uses B1 connectors; can ask for repetition; is understandable despite errors. |

## 13. Validation and Progress Model

### 13.1 Mastery States

| State | Definition | Promotion evidence |
|---|---|---|
| Not started | No meaningful exposure. | None. |
| Seen once | Student has seen input or teacher model once. | Completed introductory activity. |
| Practised with support | Student can perform with scaffolds, phrase bank, or teacher prompts. | 2 supported exercises with 60%+ accuracy. |
| Can do in controlled exercise | Student performs in a known format without heavy support. | 3 exercises at 75%+ or teacher observation at level 3. |
| Can do in mock exam | Student performs under time/pressure with mixed domains. | Mock section meets threshold; speaking/writing human-reviewed where needed. |
| Exam-ready | Stable performance across transfer tasks and official-like mock conditions. | 2 separate mock attempts above threshold; no critical missing structure/action. |

### 13.2 Evidence by Skill

| Skill | App can auto-check | AI can assist | Teacher/human must check |
|---|---|---|---|
| Writing | Word count, date/greeting/closing, required bullet presence, target vocabulary, obvious missing task parts | Grammar clarity, politeness, register, task completion, phrase coverage | Naturalness, pragmatic appropriateness, whether message sounds memorised, borderline scoring |
| Speaking | Duration, transcript, vocabulary coverage, answered question count, repair phrase use | Coherence, connector use, relevance, follow-up handling | Pronunciation, fluency, interaction, confidence, examiner-like pressure |
| Listening | MCQ/true-false/matching, exact extracted numbers/times | Diagnose missed information type and weak vocabulary | Audio difficulty calibration and dialect tolerance |
| Reading | MCQ/matching/Ja-Nein, evidence-word highlighting | Diagnose scanning vs vocabulary vs instruction misunderstanding | Ambiguous answer review, reading-strategy coaching |

### 13.3 Competency Evidence Rules

| Competency type | Evidence proving mastery | Exercise validating it | Teacher check | AI/app check |
|---|---|---|---|---|
| Extract time/date/place | 8/10 exact extractions over 2 tasks | Listening/reading detail table | Check if errors come from numbers or vocabulary | Exact match |
| Match situation to service | 4/5 correct with evidence keywords | Reading matching set | Ask student to justify selection | Answer key + evidence word |
| Write practical message | 2 timed A2 texts with all bullets | Writing task family | Tone, clarity, register | Structure, word count, bullet coverage |
| Self-introduction | 90-120 seconds, 5 prompts covered | Oral recording | Fluency, pronunciation, authenticity | Duration, transcript coverage |
| Picture-card discussion | 3-5 minutes, reason/example, follow-up | Oral scenario card | Interaction and spontaneity | Vocabulary and topic coverage |
| Swiss-German gist | 4/5 simple messages summarised correctly | Dialect recognition set | Confirm stress tolerance | Summary choice/action extraction |

## 17. Gaps, Uncertainties, and Further Verification

| Gap | Impact | Recommendation |
|---|---|---|
| No public detailed speaking/writing scoring descriptors found | App rubrics cannot be called official rubrics. | Label rubrics as `prep_extrapolation`; use teacher review for final readiness. |
